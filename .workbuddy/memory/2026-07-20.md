## TokenBase MVP — backend + payments (same project, server/ subdir)
- User wants to go live AND accept payments (foreign customers). Chose **Paddle/LemonSqueezy (MoR)** + **full MVP**.
- Built zero-dependency Node backend (server/server.js) + JSON store (server/store.js). No npm install needed → runs anywhere.
- `server/config.js` = editable prices/packages (user sets prices here: `pricing.models` per-token, `packages` top-up tiers, `freeCredit`, `upstream` keys).
- Endpoints: `/api/register` (email→sk- key, +free credit), `/api/me`, `/api/checkout` (creates Paddle/LemonSqueezy checkout or sandbox demo URL), `/api/paddle-webhook` + `/api/ls-webhook` (HMAC verify, credits balance), `/v1/chat/completions` + `/v1/messages` (proxy to upstream w/ per-token billing), `/v1/models`, `/v1/balance`.
- Payment: real mode needs PADDLE_VENDOR_ID/API_KEY/CLIENT_TOKEN/SECRET/WEBHOOK_SECRET (or LS keys) in env; **SANDBOX_DEMO=true** lets the whole loop run locally with no merchant account (checkout URL credits balance immediately).
- Upstream proxy is in **demo mode** by default (returns synthetic completion, deducts ~real token cost) until `config.upstream` keys are filled → then forwards to OpenAI/Anthropic/OpenRouter.
- Dashboard: `server/public/dashboard.html` (+ dashboard.js) — register, view key, balance, buy credits, copy endpoint snippet. Landing CTAs (index.html) now point to `/dashboard`.
- Verified full loop locally on port 3100: register→$5 credit→proxy call→balance 5→4.99995→checkout (pro $20)→sandbox pay→balance ~25.
- `server/DEPLOY.md` = deploy steps (Vercel/Netlify/Railway + supabase/neon DB), env vars, MoR setup, legal/compliance warnings (reselling OpenAI/Anthropic may violate ToS — use wholesale upstream + multi-vendor failover).
- IMPORTANT caveat to relay: this is an MVP skeleton. Before real money, wire a real DB (not JSON store), real upstream keys, real MoR webhooks, and get tax/ToS advice.
